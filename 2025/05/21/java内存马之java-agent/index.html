概念

一个运行中的 Java 程序运行在一个 JVM（Java 虚拟机）实例中。

Java Agent可以在程序运行时动态地修改Java字节码，进而动态地修改已加载或未加载的类、属性和方法的技术。

对于 Agent（代理）来讲，其大致可以分为两种，一种是在 JVM 启动前加载的premain-Agent，另一种是 JVM 启动之后加载的agentmain-Agent。这里我们可以将其理解成一种特殊的Interceptor（拦截器），如下图：

例子

假设有一个基于 Spring Boot（内嵌Tomcat） 的Web应用正在运行。攻击者的目标是：向该应用的JVM进程注入一个内存马，使得任何访问该应用的HTTP请求，只要携带了特定的秘密参数（如 password），就能在服务器上执行系统命令。

攻击者会选择修改一个每个Web请求都会经过的核心类。一个经典的目标是Tomcat的 org.apache.catalina.core.ApplicationFilterChain类的 doFilter方法，因为它是处理所有HTTP请求的过滤器链的入口

  1. 注入恶意Agent

    攻击者首先需要将包含恶意逻辑的Agent Jar包注入到目标JVM中。这通常通过一个独立的Java程序（Attach程序）完成，该程序使用 com.sun.tools.attach.VirtualMachine的Attach API

  2. Agent修改关键类字节码

    当恶意Agent被加载后，其 agentmain方法会被调用。这个方法会执行以下关键操作

    • 遍历查找目标类：使用 Instrumentation.getAllLoadedClasses()方法，在所有已加载的类中寻找 org.apache.catalina.core.ApplicationFilterChain。
    • 注册字节码转换器：通过 Instrumentation.addTransformer()方法注册一个自定义的 ClassFileTransformer。
    • 触发类重定义：调用 Instrumentation.retransformClasses(targetClass)方法。这会指示JVM对目标类进行重新转换，从而触发刚才注册的转换器的 transform方法。

  3. 植入恶意逻辑（内存马）

    在 ClassFileTransformer.transform方法中，攻击者使用 Javassist 等字节码操作库来修改 ApplicationFilterChain.doFilter方法的逻辑。他们会在该方法的开头插入恶意代码，大致内容如下

攻击效果与特点

一旦攻击成功，这个内存马就会生效：

  • 触发方式：攻击者访问 该应用上的任意URL，只要在请求参数中带上 password=预设密码&cmd=系统命令，内存马就会在服务器上执行相应的命令。
  • 高度隐蔽性：
    • 无文件落地：恶意代码存在于JVM内存中，磁盘上没有对应的WebShell文件，传统文件扫描无法检测
    • 绑定合法URL：内存马寄生在Tomcat核心组件中，不依赖特定的可疑URL路径，所有请求都会经过它
    • 驻留内存：即使应用重启，内存马也会消失，但如果攻击者能再次利用漏洞，仍可重新注入。有些高级技术可以实现持久化。

防御与检测方向

面对这种威胁，可以采取以下措施：

  • RASP（运行时应用自我保护）：在应用内部监控关键API的调用（如 Runtime.exec()），对可疑行为进行实时拦截
  • Java Agent安全监控：部署安全的Agent，监控JVM中类的重定义和重转换行为，对核心类的修改进行告警
  • 严格的访问控制与网络监控：限制服务器不必要的出网流量，对服务器上突然出现的可疑进程连接保持警惕。
